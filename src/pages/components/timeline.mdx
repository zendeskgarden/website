## How to use it

### Default

The Timeline component is made up of individual Timeline items. Item content appears after the item marker by default.

import TimelineDefault from '../../examples/accordions/timeline/TimelineDefault.tsx';
import TimelineDefaultCode from '!!raw-loader!../../examples/accordions/timeline/TimelineDefault.tsx';

<CodeExample code={TimelineDefaultCode}>
  <TimelineDefault />
</CodeExample>

### Alternating

Items are displayed on alternating sides of the Timeline.

import TimelineAlternating from '../../examples/accordions/timeline/TimelineAlternating.tsx';
import TimelineAlternatingCode from '!!raw-loader!../../examples/accordions/timeline/TimelineAlternating.tsx';

<CodeExample code={TimelineAlternatingCode}>
  <TimelineAlternating />
</CodeExample>

### Custom media

The item marker can also be replaced with another icon, image, or avatar.

import TimelineCustomMedia from '../../examples/accordions/timeline/TimelineCustomMedia.tsx';
import TimelineCustomMediaCode from '!!raw-loader!../../examples/accordions/timeline/TimelineCustomMedia.tsx';

<CodeExample code={TimelineCustomMediaCode}>
  <TimelineCustomMedia />
</CodeExample>

### Opposite content

In the opposite layout, a second content area is present on the opposite side of the Timeline.

## How to use it well

### Nest content when necessary

Components such as links, buttons, and images can be nested in each Timeline item.

<BestPractice>
  <Do imageSource={props.data.timelineNestingDo.childFile.childImageSharp.gatsbyImageData}>
    <p>Do include images and other nested content if appropriate.</p>
  </Do>
</BestPractice>

### Make interactive elements explicit

Creating a visual affordance for a link or action is preferred to making the entire item interactive.

<BestPractice>
  <Do
    imageSource={props.data.timelineInteractiveElementsDo.childFile.childImageSharp.gatsbyImageData}
  >
    <p>Create explicit visual affordances for interactive items.</p>
  </Do>
  <Dont
    imageSource={
      props.data.timelineInteractiveElementsDont.childFile.childImageSharp.gatsbyImageData
    }
  >
    <p>Don’t rely on hover states to show that items are interactive.</p>
  </Dont>
</BestPractice>

### Use correct hierarchy

Item content can be replaced with any text style. Make sure to use appropriate hierarchy when designing your content.

<BestPractice>
  <Do imageSource={props.data.timelineCorrectHierarchyDo.childFile.childImageSharp.gatsbyImageData}>
    <p>Use balanced and appropriate typographic hierarchy.</p>
  </Do>
  <Dont
    imageSource={props.data.timelineCorrectHierarchyDont.childFile.childImageSharp.gatsbyImageData}
  >
    <p>Don’t mix up typographic hierarchy when customizing item content.</p>
  </Dont>
</BestPractice>

## API

The Timeline component follows this structure:

```jsx
<Timeline>
  <Timeline.Item>
    <Timeline.OppositeContent /> {/* Optional */}
    <Timeline.Content />
  </Timeline.Item>
  {/* other items */}
</Timeline>
```
