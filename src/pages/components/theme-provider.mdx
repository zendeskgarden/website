## How to use it

### Default

The typical usage of a theme provided to styled components.

import Default from '../../examples/theme-provider/ThemeProviderDefault.tsx';
import DefaultCode from '!!raw-loader!../../examples/theme-provider/ThemeProviderDefault.tsx';

<CodeExample code={DefaultCode}>
  <Default />
</CodeExample>

### Branding

You can override the default Theme provider for a branded component
experience. In this example, Garden’s `primaryHue` is replaced and corners
are set to square.

import Branding from '../../examples/theme-provider/ThemeProviderBranding.tsx';
import BrandingCode from '!!raw-loader!../../examples/theme-provider/ThemeProviderBranding.tsx';

<CodeExample code={BrandingCode}>
  <Branding />
</CodeExample>

### Extension

The default theme can be extended with custom values. In this example, the
palette adds a new `darkOrange` hue which replaces Garden’s default
`dangerHue`.

import Extension from '../../examples/theme-provider/ThemeProviderExtension.tsx';
import ExtensionCode from '!!raw-loader!../../examples/theme-provider/ThemeProviderExtension.tsx';

<CodeExample code={ExtensionCode}>
  <Extension />
</CodeExample>

### Nesting

A nested Theme provider can access and modify parent theming values. In this
example, nesting is used to override the size, color, and shape of the
[default](#default) styled component.

import Nesting from '../../examples/theme-provider/ThemeProviderNesting.tsx';
import NestingCode from '!!raw-loader!../../examples/theme-provider/ThemeProviderNesting.tsx';

<CodeExample code={NestingCode}>
  <Nesting />
</CodeExample>
