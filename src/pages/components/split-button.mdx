<Usage>
  <Use>

    - To let users select from multiple parallel actions. Actions are parallel
    when each represents a path forward for the user and none cancel the action.
    - To reduce visual complexity when there are multiple actions a user can take

  </Use>
  <Misuse>

    - To provide a way to navigate between pages, use [Anchor](/components/anchor) instead
    - To provide a way to choose between primary and secondary actions, use two different [Buttons](/components/button) instead

## API

The SplitButton component follows this structure:

```jsx
<SplitButton>
  <Button />
  <Dropdown>
    <Trigger>
      <ChevronButton />
    </Trigger>
    <Menu>
      <Item />
      <Item />
      {/* etc. */}
    </Menu>
  </Dropdown>
</SplitButton>
```

**Implementation notes**

The split button pattern contains:

- [SplitButton](#splitbutton) component as a container
- [Button](#button) component for the main action
- [ChevronButton](#chevronbutton) component to show/hide secondary actions
- [Dropdown](/components/menu/#dropdown), [Menu](/components/menu/#menu) and
  [Trigger](/components/menu/#trigger) components for the secondary actions menu
