## API

The Accordion component follows this structure:

```jsx
<Accordion>
  <Accordion.Section>
    <Accordion.Header>
      <Accordion.Label />
    </Accordion.Header>
    <Accordion.Panel />
  </Accordion.Section>
  {/* other sections */}
</Accordion>
```

### Accordion

<Component components={props.data.mdx.components} componentName="accordion" />

The Accordion component provides state and accessibility attributes to its children. It follows the
[W3C Accordion pattern](https://www.w3.org/TR/wai-aria-practices/#accordion).

<PropSheet components={props.data.mdx.components} componentName="accordion" />

#### Accordion.Header

<Component components={props.data.mdx.subcomponents} componentName="accordion.header" />

A Header acts as the trigger for the corresponding [Panel](#accordionpanel) to be expanded or collapsed. It
renders an icon to show the current expansion state. Nest it within a [Section](#accordionsection) component.

<PropSheet components={props.data.mdx.subcomponents} componentName="accordion.header" />

#### Accordion.Label

<Component components={props.data.mdx.subcomponents} componentName="accordion.label" />

Nest the Label within a [Header](#accordionheader) component.

<PropSheet components={props.data.mdx.subcomponents} componentName="accordion.label" />

#### Accordion.Panel

<Component components={props.data.mdx.subcomponents} componentName="accordion.panel" />

Nest a Panel within a [Section](#accordionsection) component.

<PropSheet components={props.data.mdx.subcomponents} componentName="accordion.panel" />

#### Accordion.Section

<Component components={props.data.mdx.subcomponents} componentName="accordion.section" />

Wrap each section's [Header](#accordionheader) and corresponding [Panel](#accordionpanel) together with
the Section component.

<PropSheet components={props.data.mdx.subcomponents} componentName="accordion.section" />
