A button is an element on a page that tells a user what they can do and
triggers an action. They may be used for:

- submitting information
- beginning a new task
- triggering a new element to appear on the page
- specifying the next step in a process

## Anatomy of Buttons

There is one call-to-action (CTA) per button, but there may be a few buttons
a page or view. Buttons in the right place and with clear labels can help
people feel confident they are taking the correct action.

![Button types](figma:patterns-buttons-anatomy-button-type.png)

1. **Primary** buttons are for the most important action in a given view
1. **Default** buttons are for secondary actions
1. **Basic** buttons are for repetitive or deemphasized actions in a given view

## How Buttons look

Button size plays a part in content and user interaction. Clear labels and
visual signifiers such as shape, color, and size help communicate the types
of actions available and their priority in the process.

- The default button size is medium
- Use small buttons on screens that have more information density
- Large buttons are mostly used in onboarding

## How Buttons sound

The button CTA should guide the experience, not just finish a page.
Typically, the CTA repeats the page heading when possible (e.g. “Add agent”).
The phrase repetition confirms the action that will take place.

Use the CTA order of `Verb noun` (such as “Upload file”) to clearly
communicate the action. Omit articles, pronouns, and punctuation. If multiple
actions are changing on the page or screen, “Save changes” is a good option.

See below the difference between clearly labeled buttons and buttons with
vague, general labels.

<BestPractice>
  <Do imageSource={props.data.buttonsContentDo.childFile.childImageSharp.gatsbyImageData}>
    <ul>
      <li>Communicate a single action per button</li>
      <li>Use the “Verb noun” formula</li>
      <li>Keep buttons in sentence case</li>
      <li>Consider longer languages. Use 1–4 words and a maximum of 20 characters.</li>
    </ul>
  </Do>
  <Dont imageSource={props.data.buttonsContentDont.childFile.childImageSharp.gatsbyImageData}>
    <ul>
      <li>Avoid CTAs that lack context like "Yes," "No," or "Done"</li>
      <li>Avoid calling out more than one action</li>
      <li>Avoid ALL CAPS</li>
      <li>Avoid articles, pronouns, and punctuation</li>
    </ul>
  </Dont>
</BestPractice>

## Buttons in action

**Loading** buttons display a [Dots loader](/components/dots) inside the
button to show loading progress. The Dots loader appears after the user has
initiated an action, signifying the site is loading the next action to occur.
Buttons shouldn’t change size while loading.
